# 📰 Daily Digest Bot — 개인 맞춤 뉴스 요약 텔레그램 봇

> 매일 아침 8시, 내가 구독하는 뉴스레터·유튜브(나중에는 인스타까지)의 새 소식을
> 한국어로 요약해서 텔레그램으로 보내주는 개인 비서 봇.

작성일: 2026-04-23
작성자: 이원영 × Claude (인터뷰 기반)

---

## 1. 한 줄 요약 (What are we building?)

> **"내가 지정한 채널들의 어제~오늘 새 콘텐츠를 모아서, Claude가 한국어 한 줄씩 요약하고, 매일 아침 8시 텔레그램으로 보내주는 자동 봇."**

## 4. 전체 동작 흐름 (How it works)

```
┌─────────────────────────────────────────────────────────────────┐
│                 매일 아침 07:30 KST (22:30 UTC)                  │
│                   ⏰ Cron Trigger 자동 실행                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ① 수집 (Fetcher)                                                │
│    • 뉴스레터: Gmail 전용 주소 inbox 읽기 (IMAP/Gmail API)        │
│    • 유튜브: 채널별 최신 영상 조회 (YouTube Data API)             │
│    → 어제 00:00 ~ 오늘 07:30 사이 새 항목만 수집                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ② 중복 제거 (Deduplicator)                                      │
│    • URL 해시를 KV 저장소에서 조회                                │
│    • 이미 본 항목 걸러냄 · 30일 넘은 기록 자동 정리                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ③ 본문 추출 (Extractor)                                         │
│    • 뉴스레터: 이메일 본문 추출                                   │
│    • 유튜브: 자막 API로 transcript 가져오기                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ④ 요약 + 번역 (Summarizer)                                      │
│    • Claude API (Haiku 기본, 긴 영상은 Sonnet)                   │
│    • 한 줄 요약 (30~50자) + 필요시 한국어 번역                    │
│    • Provider 인터페이스로 추상화 → GPT/Gemini 교체 가능          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ⑤ 포맷팅 (Formatter)                                            │
│    • 카테고리별 그룹핑 (📧 뉴스레터 / 📺 유튜브)                  │
│    • Telegram MarkdownV2 형식으로 메시지 조립                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  ⑥ 전송 (Sender)                                                 │
│    • Telegram Bot API · 08:00 KST 정각 발송                     │
│    • 요약 결과는 DB에 저장 (검색용 · 30일 보관)                   │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 기술 스택 (선택 이유 포함)

| 영역 | 선택 | 이유 |
|---|---|---|
| **런타임** | TypeScript + Cloudflare Workers | 무료 티어 충분, Cron Trigger 내장, KV/D1 포함 |
| **스케줄러** | Cloudflare Cron Trigger | 별도 서버 불필요, UTC 기준 분 단위 스케줄 |
| **이메일 수집** | Cloudflare Email Routing + Workers | 전용 주소로 뉴스레터 받으면 Worker가 즉시 처리 가능 |
| **유튜브 수집** | YouTube Data API v3 + `youtube-transcript` | 공식 API (무료 쿼터 넉넉) + 자막으로 요약 품질↑ |
| **LLM** | Anthropic Claude API (Haiku) | 인터뷰 확정. Provider 추상화로 교체 가능 |
| **저장소** | Cloudflare KV (중복·구독) + D1 (요약 아카이브) | 둘 다 Workers 기본 제공, 무료 |
| **봇** | Telegram Bot API (`node-telegram-bot-api`) | 무료, 간단 |
| **설정 파일** | `sources.yaml` (리포 내 파일) | Claude가 자연어 요청 받고 직접 편집 |

### 왜 Vercel이 아니라 Cloudflare Workers?
- Cloudflare는 Email Routing으로 **전용 이메일 주소 → Worker 호출**이 바로 가능 (뉴스레터 수집에 최적)
- Vercel Cron도 가능하지만, 이메일 수집은 별도 IMAP 폴링 필요해서 더 복잡
- 둘 다 무료 티어지만 이 프로젝트엔 Cloudflare가 궁합 좋음

## 6. 데이터 구조

### 6.1 `sources.yaml` (구독 목록)
```yaml
newsletters:
  - name: "Ben's Bites"
    email_from: "newsletter@bensbites.co"
    language: "en"
  - name: "디독"
    email_from: "didoc@stibee.com"
    language: "ko"

youtube:
  - name: "조코딩"
    channel_id: "UCQNE2JmbasNYbjGAcuBiRRg"
    language: "ko"
  - name: "Fireship"
    channel_id: "UCsBjURrPoezykLs9EqgamOA"
    language: "en"
```

### 6.2 KV Store (Cloudflare KV)
- **중복 체크**: `seen:<sha256(url)>` → `{date: "2026-04-22"}` (TTL 30일)
- **구독 상태 캐시**: `youtube:last_video_id:<channel_id>` → 마지막 영상 ID

### 6.3 D1 DB (요약 아카이브 · 검색용)
```sql
CREATE TABLE summaries (
  id INTEGER PRIMARY KEY,
  date TEXT NOT NULL,          -- 2026-04-23
  source_type TEXT,            -- 'newsletter' | 'youtube'
  source_name TEXT,            -- '조코딩'
  title TEXT,
  summary_ko TEXT,             -- 한국어 요약 (검색 대상)
  url TEXT,
  created_at INTEGER
);
CREATE INDEX idx_date ON summaries(date);
CREATE VIRTUAL TABLE summaries_fts USING fts5(title, summary_ko);
```

## 7. MVP 구현 순서 (우선순위 + 예상 시간)

> **목표**: "뉴스레터 + 유튜브 → 아침 알림" 1차 완성

### Phase 1 — 뼈대 만들기 (하루)
- [ ] Cloudflare Workers 프로젝트 초기화 (`wrangler init`)
- [ ] Cron Trigger 테스트 (매일 22:30 UTC = KST 07:30 발송)
- [ ] Telegram 봇 생성, 토큰 발급, 내 chat_id로 테스트 메시지 성공
- [ ] `sources.yaml` 스켈레톤 작성

### Phase 2 — 유튜브 수집 파이프라인 (1~2일)
- [ ] YouTube Data API 키 발급
- [ ] 채널 ID로 최근 24시간 영상 목록 가져오기
- [ ] 자막 추출 → Claude Haiku로 한 줄 요약 (영어→한국어 포함)
- [ ] 중복 체크 (KV 기반)
- [ ] 포맷팅해서 텔레그램 발송

### Phase 3 — 뉴스레터 수집 파이프라인 (1~2일)
- [ ] Cloudflare Email Routing 설정 (예: `digest@내도메인`)
- [ ] 구독 주소 → 이 이메일로 전환 or 자동 전달 설정
- [ ] Email Worker로 수신 이메일 저장 → Cron 시점에 일괄 요약
- [ ] 동일하게 요약·발송 플로우 연결

### Phase 4 — 요약 아카이브 + 검색 (하루)
- [ ] D1 DB 생성 · 요약 저장
- [ ] Telegram 봇에서 `/검색 <키워드>` 명령 처리
- [ ] FTS5로 풀텍스트 검색 → 최근 20건 반환

### Phase 5 — 다듬기 (하루)
- [ ] 에러 알림 (요약 실패 시 관리자 텔레그램으로 에러 로그)
- [ ] 30일 지난 데이터 자동 정리 크론
- [ ] README 업데이트 · 배포

**예상 총 소요**: 개발 집중 시 1주일 내외 (빈도와 숙련도에 따라 변동)

| 리스크 | 영향 | 대응 |
|---|---|---|
| 유튜브 자막이 없는 영상 | 요약 불가 | 제목+설명만으로 한 줄 생성, 또는 Whisper 호출 (비용↑) |
